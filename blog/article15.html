OS Credential Dumping

MITRE ID: T1003 Tactic: Credential Access Published: October 20, 2025 Author: Mahamad Sekh

Technique Overview

Adversaries attempt to dump credentials from operating system memory or security stores to obtain account login and credential material. Common targets include LSASS memory, SAM database, and cached domain credentials.

How OS Credential Dumping Works

This credential access technique is commonly observed in sophisticated attack campaigns where adversaries seek to steal authentication credentials. Understanding the mechanics of this technique is crucial for security teams implementing effective detection and prevention strategies.

Key Characteristics

  • Part of the MITRE ATT&CK Credential Access tactic
  • Technique ID: T1003
  • Commonly used in advanced persistent threat (APT) campaigns
  • Requires security monitoring and threat detection capabilities

Detection Strategies

Monitor for suspicious process access to LSASS.exe, unusual use of credential dumping tools (Mimikatz, ProcDump), registry access to SAM/SECURITY hives, and abnormal memory read operations.
